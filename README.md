# Практическая работа по Django

Мини-приложение для управления пользователями, организациями и должностями.

## Функционал (Практическая работа №1)
- CRUD для организаций  
- CRUD для должностей  
- Связь пользователей с организациями на определённой должности  
- Один пользователь может состоять в нескольких организациях  
- В одной организации — только одна должность на пользователя  
- Дефолтное значение в должности — **"ассистент преподавателя"**

## REST API (Практическая работа №2)
Реализован RESTful API для основных сущностей с использованием **Django REST Framework**:

### Эндпоинты:
- **`GET /api/users/`** — список пользователей с указанием организаций, в которых они состоят  
- **`GET /api/users/{id}/`** — детальная информация о пользователе  
- **`/api/organizations/`** — полный CRUD для организаций (создание, чтение, обновление, удаление)  
- **`GET /api/organizations/many-employees/`** — **дополнительное действие**:  
  возвращает **топ-10 организаций** с наибольшим числом сотрудников

### Особенности:
- Использованы `ModelViewSet` и `ReadOnlyModelViewSet`  
- Для специального эндпоинта применён декоратор `@action`  
- Маршрутизация реализована через `SimpleRouter`  
- Все данные возвращаются в формате **JSON**
